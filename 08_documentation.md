# 論文、研究紀錄與文件撰寫提示詞

適合撰寫研究紀錄簿、論文、以及專案的 GitHub README 或系統教學。

---

## 1. 論文 / 研究紀錄專用

```text
【模式：研究論文寫作模式】

請以 IEEE / SCI 論文寫作標準協助我撰寫內容。

請遵守：

1. 使用正式學術語言。
2. 避免口語化描述。
3. 必須包含理論基礎。
4. 必須說明設計動機。
5. 必須描述方法細節。
6. 必須分析預期效果或研究意義。
7. 若涉及模型或演算法，請提供數學表示。
8. 若資訊不足，請標註假設，不可編造結果。

請依照以下結構輸出：

1. 研究主題
2. 研究動機與背景
3. 問題定義
4. 理論基礎
5. 方法設計
6. 系統流程
7. 預期結果
8. 研究貢獻
9. 限制與未來方向

語氣需正式、嚴謹、適合直接放入論文或研究報告。
```

## 2. GitHub README / 文件專用

```text
【模式：技術文件撰寫模式】

請以專業技術文件工程師角度協助我撰寫文件。

文件需符合：

1. 讓第一次看到專案的人能理解用途。
2. 讓開發者能快速安裝與執行。
3. 讓使用者能知道輸入、輸出與限制。
4. 讓維護者能理解架構與資料流。
5. 讓未來的我能重現此流程。

請包含：

1. 專案簡介
2. 功能特色
3. 系統架構
4. 環境需求
5. 安裝方式
6. 使用方式
7. 專案結構
8. 設定檔說明
9. 常見問題
10. 開發注意事項
11. 未來擴充方向

請避免：
- 過度行銷語氣。
- 只有指令沒有說明。
- 缺少環境版本。
- 缺少輸入輸出說明。
- 缺少錯誤排除方式。
```
