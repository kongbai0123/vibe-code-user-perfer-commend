# 情境式 AI 提示詞庫 (Prompt Profiles)

這是一個專為開發與工程任務打造的 **「情境式 AI 提示詞庫」**。
不同任務不該使用同一套 prompt，而是根據實際情境切換 AI 的「工作模式」，以確保產出的程式碼與架構符合專業的工程標準。

## 核心概念

```text
AI 角色模式 + 工程原則 + 禁止事項 + 輸出格式
```

## 檔案分類用途

| 檔案名稱 | 適用情境與用途 |
| :--- | :--- |
| `01_master_rules.md` | **總控與防暴走原則**：適用所有情境，最高優先級。包含「防暴走檢查」與「最小變更原則」，確保 AI 不會亂改程式或亂出爛 code。 |
| `02_vibe_coding.md` | **Vibe Coding 快速開發**：適合快速 prototype、AI 生產程式碼與功能迭代。已融合過往的工程設計與可維護性原則。 |
| `03_refactor_and_review.md` | **重構與審查**：適合整理既有程式碼、模組化、減少耦合，或是要求 AI 用資深審查者角度做 Code Review。 |
| `04_debug_diagnosis.md` | **除錯與診斷**：當程式壞掉、模型輸出怪異、系統不穩時，強制 AI 使用系統化除錯，而不是亂猜答案。 |
| `05_architecture_design.md` | **架構設計**：適合開始一個新專案、新功能或新系統時，要求 AI 優先考慮分層與資料流。 |
| `06_algorithm_and_ai.md` | **演算法設計與模型訓練**：針對機器學習、YOLO、CNN、光流、控制演算法等，要求明確的數學模型、實驗設計與驗證指標。 |
| `07_edge_and_control.md` | **邊緣部署與機電控制**：針對 Jetson、嵌入式系統部署，或是馬達控制、感測器融合等硬體整合場景，強調效能、限制與安全性。 |
| `08_documentation.md` | **論文與文件撰寫**：適用於 IEEE / SCI 論文寫作、GitHub README 或專案開發技術紀錄。 |
| `09_prompt_profiles.md` | **推薦組合與配置檔**：提供日常開發中最推薦的 prompt 搭配組合，以及極簡版萬用提示詞。 |

## 使用方式

在與 AI 開始新對話時，根據你的任務目標，挑選適合的 markdown 檔案內容，複製並作為系統提示詞（System Prompt）或是對話的第一句話。
例如：
> 「請套用 `04_debug_diagnosis.md` 的模式。以下是我的錯誤訊息...」
